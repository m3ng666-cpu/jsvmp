# secure-lite 设计稿（兼容优先）

## 目标

`secure-lite` 不是高强度对抗模式，而是兼容优先的发布档位：

- 先保证加密后 JS 可运行
- 先保证旧接口和现有调用不回退
- 先控制性能和体积恶化
- 在此前提下做有限的结构去规则化

## 范围

本阶段只允许：

- seed 驱动的稳定构建差异
- 轻量外壳去特征化
- 常量与字符串的温和整理
- 不改变运行语义的包装层调整

本阶段不进入：

- 重型 anti-debug
- 激进运行时绑定
- 高风险控制流重写
- 明显影响兼容性的深层 VM 结构改造

## 验收

- 兼容样本全部通过
- `/api/js/jsString` 冒烟通过
- 旧 `/js/*` 路径不回退
- 默认 8080 服务可运行
